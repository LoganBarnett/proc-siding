* TODO Track and expose Prometheus metrics for pressure transitions and action failures

Forward-port the Prometheus exporter pattern from ~rust-template~.  The template
uses the ~prometheus~ crate (0.13) with ~Registry~, ~IntCounter~, and
~TextEncoder~.

proc-siding is not a web server, so expose metrics via a minimal HTTP listener
on a configurable address (e.g. ~metrics_listen = "127.0.0.1:9191"~; disabled
when absent).

Metrics to expose:

- ~proc_siding_pressure_transitions_total~ (counter) — times pressure was
  detected and actions fired.
- ~proc_siding_clear_transitions_total~ (counter) — times pressure cleared and
  actions fired.
- ~proc_siding_action_errors_total~ (counter, label: ~action~) — action
  failures, labeled by action description.
- ~proc_siding_gpu_pressure_ratio~ (gauge) — most recent sampled pressure
  value.

Implementation steps:

1. Add ~prometheus = "0.13"~ to workspace dependencies.
2. Add a ~metrics.rs~ in ~crates/lib/src/~ defining a ~Metrics~ struct holding
   the registry and counters/gauges, with a constructor that registers them.
3. Thread ~Metrics~ into ~Monitor~ and bump counters on transitions and action
   errors.
4. Add a ~metrics_server.rs~ in ~crates/lib/src/~ that binds a TCP listener and
   serves ~/metrics~ using ~TextEncoder~.  Keep it simple — raw
   ~std::net::TcpListener~ in a spawned thread is fine; no need for axum.
5. Add ~metrics_listen: Option<String>~ to ~AppConfig~.  When set, spawn the
   metrics server thread in ~main.rs~.
6. Add integration tests: start the metrics server, trigger transitions via a
   mock monitor, scrape ~/metrics~, and assert counter values.
